# Development Rules

## Python

- **Типизация**: строгие аннотации типов (Type Hints) во всём коде
- **Асинхронность**: весь I/O — `async/await`. Никаких `time.sleep`, синхронных HTTP.
- **Обработка ошибок**: `try/except` при работе с внешними API
- **print() запрещён**: только `logger` (logging.getLogger)

## Архитектура

- **MetaOrchestrator** — единственная точка входа. Никакого классификатора, никаких веток в графе.
- **Скиллы** — единственный способ добавить новое поведение. Регистрация через `register_skill()`.
- **Кодовая логика** не должна дублироваться между скиллами. Общие функции выносить в `utils.py` или общие модули.
- **write_file** — только для создания новых файлов. Для изменений существующего кода → `apply_patch`.
- **Q&A** — MetaOrchestrator отвечает сам с read/write/bash инструментами. Отдельного QA-скилла нет.

## LangGraph

- Граф — один узел `meta_orchestrator` → END
- `AgentState` содержит только `input_content`, `final_response`, `error`
- Никаких промежуточных узлов, условных рёбер, ToolNode

## Prompts

- Промпты хранятся в `prompts/` в формате `.md`
- `system/system.md` — базовая личность и тон агента
- `agents/coordinator.md` — роль координатора + список скиллов (загружается после system.md)
- Каждый промпт может использовать шаблонные переменные `{var}` через `str.format()`

## Безопасность

- Все bash-команды проходят через Security Agent Interceptor (`src/agent/security.py`)
- Заблокированные команды не выполняются, возвращается сообщение об ошибке

## Логирование

- Все LLM-вызовы и tool-вызовы логируются в JSONL (`logs/{session_id}.jsonl`)
- Использовать `logger` из модуля `src.agent.logger`

## Хранилище

- `raw/`, `wiki/`, `memory/` — не хранятся в git (кроме `.gitkeep`)
- `schema/index.json` — в `.gitignore`
- ID статей — `art_{uuid12}`, вики-файлы — `{title}_{uuid8}.md`
- Перекрёстные ссылки между статьями — автоматические через теги

## Правило Writeback

- Любое архитектурное изменение → обновить `docs/ARCHITECTURE.md`
- Новый скилл → обновить `docs/PROJECT_MAP.md`
- Новый инструмент/промпт → обновить соответствующий документ
