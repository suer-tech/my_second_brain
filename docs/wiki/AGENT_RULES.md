Это дополнение к глобальным правилам из [CLAUDE.md](../../CLAUDE.md).

## Разработка на Python
1. **Типизация**: Строго используй `Type Hints` (аннотации типов) во всем коде.
2. **Асинхронность**: Весь код бота и вызовы к API должны быть асинхронными (`async/await`). Избегай блокирующих вызовов `time.sleep` или синхронных HTTP-запросов (используй `aiohttp` или асинхронные SDK).
3. **Обработка ошибок**: Обязательно используй `try/except` при работе с внешними API (YouTube, DeepSeek).
4. **Конфигурация**: Секреты (API ключи) должны лежать в `.env` файле и загружаться через `pydantic-settings` или `os.getenv`.

## Правило Writeback
- Если ты изменяешь логику (например, добавляешь новый узел в LangGraph в файле `src/agent/graph.py`), ты **ОБЯЗАН** зайти в `ARCHITECTURE.md` и обновить блок с описанием "Поток данных".
- Любая новая сущность базы данных или структуры `wiki/` должна быть описана в документации до/во время ее реализации.
