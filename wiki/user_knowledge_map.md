# Профиль пользователя и Карта знаний (Knowledge Map)

Этот файл является динамической картой компетенций пользователя в сфере AI-агентной разработки. Агенты должны опираться на этот файл при генерации ответов и выжимок из новых статей.

## Базовый профиль
- **Роль**: AI Architect / Technical PM (фокус на архитектуре, а не написании кода руками).
- **Формат обучения**: При анализе новых статей/документов агент должен выдавать **комбинацию**: сначала короткое бизнес-саммари (суть, польза, косты), затем техническая архитектура (Mermaid-схемы, паттерны).

## Освоенные компетенции (Что пользователь уже знает)
Агенту **НЕ НУЖНО** объяснять эти концепции с нуля:
- **Многоагентные системы (Multi-agent Systems)**: Есть практический опыт проектирования.
- **Function Calling / Tool Use**: Глубокое понимание механики работы инструментов.
- **ReAct (Reason + Act)**: Понимание базовых цепочек рассуждений LLM.
- **Парадигма работы с контекстом**: Пользователь осознает, что классический векторный RAG устаревает. Будущее за агентами, которые автономно ориентируются по графам внутри документации проекта (LLM-Wiki).

## Зоны роста (Фокус для обучения и Q&A)
Агент должен делать **акцент** на этих темах при анализе новых материалов:
1. **Продвинутое проектирование графов (LangGraph/StateGraph)**
   - Управление сложными состояниями (State management).
   - Маршрутизация (Routing, Conditional Edges) и Error-Handling.
2. **Оценка агентов (Evaluation & Testing)**
   - Защита от галлюцинаций в многошаговых цепочках.
   - LLM-as-a-Judge, фреймворки для оценки и тестирования метрик.
3. **MLOps и Оптимизация (Cost & Speed)**
   - Снижение костов через OpenRouter (Flash vs Pro).
   - Кеширование (Semantic Cache).
